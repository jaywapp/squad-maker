# 스쿼드 메이커 UI/UX 점검 보고서

- 점검일: 2026-07-18
- 점검 대상: `main` (`d7f4da2`, 반응형 UX 개편 반영본)
- 점검 범위: 스쿼드 편집, 전술 패턴, 매치 전략, 공유 뷰어, 모바일/데스크톱 반응형, 접근성, 데이터 안전성
- 점검 방법: 코드·설계 문서 검토, GitHub Pages 라이브 데모를 1280px 및 390px 뷰포트에서 직접 조작·확인

## 1. 종합 평가

이번 개편으로 제품의 기본 골격은 크게 좋아졌다. 데스크톱 2패널은 필드를 보면서 지침을 작성할 수 있고, 390px 모바일에서도 필드가 가로 스크롤 없이 온전히 들어온다. 전술 보드 테마는 기능의 성격과 잘 맞으며, 공유 링크를 별도 읽기 전용 모드로 분리한 결정도 데이터 보호와 열람 경험 모두에 효과적이다.

현재 남은 개선점은 시각 스타일보다 **작업 안전성, 모바일 정보 밀도, 숨은 제스처의 발견 가능성, 키보드·스크린리더 접근성**에 집중되어 있다. 치명적인 레이아웃 문제는 발견하지 못했지만, 사용자가 실제 경기 준비를 위해 데이터를 많이 입력할수록 불편이 커지는 구조가 남아 있다.

### 현재 UX 성숙도

| 영역 | 평가 | 요약 |
|---|---:|---|
| 시각적 일관성 | 4/5 | 전술 보드 콘셉트와 3계열 버튼 체계가 안정적 |
| 반응형 레이아웃 | 4/5 | 390px 필드 적응과 2패널 전환이 정상 동작 |
| 핵심 작업 효율 | 3/5 | 모바일 지침 입력과 패턴 편집에서 스크롤·버튼 밀도가 높음 |
| 발견 가능성 | 2.5/5 | 더블클릭·우클릭·롱프레스 등 숨은 제스처 의존도가 높음 |
| 오류 예방·복구 | 2.5/5 | 확인 모달은 있으나 삭제/변경 후 되돌리기가 없음 |
| 접근성 | 2.5/5 | 포커스 링과 ARIA 기초는 좋지만 모달·탭·색상 대비 보완 필요 |

## 2. 잘된 점

1. **데스크톱 편집 흐름 개선**
   - 필드가 왼쪽에 고정되고 오른쪽에서 팀 설정과 지침을 편집할 수 있어 시선 왕복이 짧다.
   - 1280px 환경에서 필드와 편집 패널이 동시에 보이며, 기존 단일 컬럼의 가장 큰 병목이 해소됐다.

2. **모바일 필드 적응**
   - 390px 환경에서 필드가 약 355px 폭으로 맞춰지고 가로 넘침이 발생하지 않았다.
   - 선수 번호와 이름, 필드 라인이 작은 화면에서도 식별 가능했다.

3. **공유 뷰어 분리**
   - 편집 전용 컨트롤이 노출되지 않고 `내 스쿼드로 가져오기`를 명시적으로 선택해야 저장된다.
   - 열람자가 공유 링크를 열었을 때 기존 로컬 스쿼드가 자동으로 덮어써지지 않는 구조가 적절하다.

4. **터치 타깃 개선**
   - 모바일 전술 패턴의 이전/다음, 추가/삭제, 재생/GIF 버튼이 모두 최소 44px 높이를 충족했다.

5. **피드백 기반 UI**
   - 네이티브 `confirm`을 자체 다이얼로그로 교체했고, 복사·추가·삭제 등 주요 동작에 토스트를 제공한다.
   - 포메이션 및 인원 변경 전 데이터 초기화를 명시적으로 경고한다.

## 3. 우선순위별 개선 제안

## P1 — 다음 개선 사이클 권장

### 3.1 되돌리기 없는 파괴적 작업

**관찰**

- 선수, 패턴, 단계, 이동 경로 삭제는 즉시 상태에 반영되고 자동 저장된다.
- 포메이션·인원 변경에는 확인창이 있지만 실행 후 복구할 방법은 파일 백업 외에 없다.
- 모바일에서는 잘못 탭할 가능성이 높아 확인창만으로는 충분하지 않다.

**제안**

- 최근 20개 상태를 보관하는 가벼운 undo/redo 스택을 도입한다.
- 최소 범위로 시작한다면 삭제 직후 5초간 `선수 삭제됨 · 되돌리기` 액션 토스트를 제공한다.
- 1차 적용 대상: 선수 삭제, 패턴 삭제, 단계 삭제, 경로 삭제, 포메이션 변경, 인원 변경.
- 키보드에서는 `Ctrl/Cmd+Z`, `Ctrl/Cmd+Shift+Z`를 지원한다.

**기대 효과**: 입력 손실 불안 감소, 실수 복구 시간 단축, 자동 저장에 대한 신뢰 향상.

### 3.2 모바일 지침 입력의 과도한 세로 길이

**관찰**

- 9vs9 기본 상태에서 모바일 스쿼드 화면 전체 높이는 약 2,025px이다.
- 개별 선수 9명의 입력란이 모두 펼쳐져 있어 특정 선수 지침을 찾기 위해 긴 스크롤이 필요하다.
- 11vs11에서는 이 문제가 더 커진다.

**제안**

- 개별 선수 지침을 기본 접힘 상태의 아코디언 또는 선수 선택 방식으로 바꾼다.
- `지침 있음`, `미작성` 필터와 작성 개수(`3/9 작성`)를 제공한다.
- 모바일에서는 한 번에 한 선수만 펼치고, 이전/다음 선수 탐색을 지원한다.
- 전체 지침 카드 상단에 `개별 지침 작성` 진입 버튼을 두어 필드 편집과 텍스트 편집을 단계적으로 분리한다.

**기대 효과**: 모바일 스크롤 40~60% 감소, 특정 선수 재편집 속도 향상.

### 3.3 선수 편집이 숨은 제스처에 의존

**관찰**

- 이름 수정은 더블클릭, 색상·삭제는 우클릭/롱프레스에 의존한다.
- 힌트는 0.72rem의 작은 보조 텍스트여서 초심자가 놓치기 쉽다.
- 필드 선수 요소는 키보드 포커스 대상이 아니어서 키보드만으로 편집할 수 없다.

**제안**

- 선수 단일 탭/클릭 시 선택 상태를 표시하고, 필드 하단에 `이름 · 색상 · 지침 · 삭제` 액션 바를 노출한다.
- 기존 더블클릭/우클릭/롱프레스는 빠른 단축 동작으로 유지한다.
- 선수 요소에 명확한 접근 가능한 이름, `tabindex="0"`, Enter/Space 메뉴 열기를 제공한다.
- 첫 방문에만 2~3단계 코치마크를 제공하거나, 힌트에 `선수를 눌러 편집`을 우선 노출한다.

**기대 효과**: 학습 비용 감소, 터치·키보드 사용성 개선.

### 3.4 전술 패턴 모바일 편집의 작업 순서 불명확

**관찰**

- 모바일에서 패턴 이름, 패턴 탐색/추가/삭제, 단계 탐색/추가/삭제, 재생, 초기화, GIF 저장 등 12개 버튼이 연속 노출된다.
- 필드 아래에서 삭제 버튼이 단독 줄로 밀리는 등 그룹의 시각적 균형이 깨질 수 있다.
- 사용자가 `패턴 선택 → 단계 편집 → 미리보기 → 내보내기` 순서를 스스로 추론해야 한다.

**제안**

- 컨트롤을 `패턴`, `단계`, `재생·내보내기` 3개 그룹으로 명시한다.
- 모바일에서는 필드 하단에 현재 단계와 `이전/재생/다음`만 고정하고, 나머지는 `편집 옵션` 바텀시트로 이동한다.
- 삭제는 각 그룹의 더보기 메뉴 안으로 이동하고, 주 동작과 시각적으로 분리한다.
- 빈 패턴에는 필드 위에 `선수나 공을 드래그해 1단계를 만드세요`라는 직접적인 시작 안내를 표시한다.

**기대 효과**: 초회 성공률 상승, 오조작 감소, 작은 화면에서 핵심 동작 집중.

### 3.5 모달·토스트 접근성 보완

**관찰**

- 확인 다이얼로그는 포커스 트랩과 포커스 복귀가 구현되어 있다.
- 반면 공유 모달과 선수 시점 모달은 열릴 때 포커스 이동, 포커스 트랩, 닫힌 후 트리거 복귀가 없다.
- 토스트는 시각적으로만 추가되며 `role="status"` 또는 `aria-live`가 없다.

**제안**

- 공유/선수 모달도 공통 Dialog 유틸리티를 사용해 포커스 이동·트랩·복귀를 통일한다.
- 모달이 열릴 때 배경에 `inert`를 적용하고, 제목을 `aria-labelledby`로 연결한다.
- 토스트 컨테이너에 `role="status" aria-live="polite"`를 적용한다.
- 닫기 버튼은 모바일에서도 최소 44×44px로 유지한다.

**기대 효과**: 키보드·스크린리더 사용 가능성 향상, 접근성 회귀 방지.

### 3.6 일부 텍스트 색상 대비 부족

**관찰**

- `--faint`(`#5c6b61`)와 배경(`#141a16`)의 대비는 약 **3.14:1**로, 작은 플레이스홀더·보조 텍스트에 부족하다.
- 기본 스쿼드 탭의 흰 글자 대비는 약 **4.32:1**, 공격 스쿼드는 약 **3.83:1**이다.
- 일반 크기 텍스트의 권장 기준 4.5:1에 미달한다.

**제안**

- `--faint`를 최소 `#74847a` 수준으로 올리거나 작은 텍스트에는 `--muted`를 사용한다.
- 기본/공격 탭 배경을 더 어둡게 조정하거나 밝은 배경에는 어두운 텍스트를 사용한다.
- CI 또는 수동 체크리스트에 주요 토큰 대비 검사를 추가한다.

**기대 효과**: 야외·저밝기 화면에서 가독성 향상, WCAG AA 대응 범위 확대.

## P2 — 품질 고도화

### 3.7 공유 뷰어의 상단 행동 문구 정리

**관찰**

- `내 스쿼드로 가져오기`와 `내 스쿼드 열기`는 차이를 한 번에 이해하기 어렵다.
- 두 버튼이 비슷한 시각적 무게를 가져 우선순위도 모호하다.

**제안**

- 문구를 `이 전술을 내 편집본으로 복사`와 `내 기존 스쿼드로 돌아가기`로 구체화한다.
- 기존 데이터를 덮어쓰는 방식보다 공유 전술을 임시 슬롯으로 복제한 뒤 저장 여부를 선택하게 하는 방식이 안전하다.
- 공유 뷰어의 기본 목적은 열람이므로 두 액션은 보조 영역 또는 더보기 메뉴로 낮춘다.

### 3.8 탭 키보드 패턴 구현

**관찰**

- 앱 모드와 스쿼드 선택에 `role="tab"`이 적용되어 있으나 모든 탭이 기본 탭 순서에 남아 있다.
- 방향키, Home/End 이동과 roving `tabindex`가 없다.

**제안**

- 선택된 탭만 `tabindex="0"`, 나머지는 `-1`로 설정한다.
- 좌/우 방향키와 Home/End로 탭을 이동하고 선택 상태를 동기화한다.
- 가능하다면 각 탭과 패널을 `aria-controls`/`aria-labelledby`로 연결한다.

### 3.9 자동 저장 상태 가시화

**관찰**

- 입력은 자동 저장되지만 사용자는 저장 완료 여부를 직접 확인할 수 없다.
- 공유 전술과 내 로컬 전술이 공존하므로 현재 데이터 출처도 중요하다.

**제안**

- 워드마크 근처에 `저장 중…` → `방금 저장됨` 상태를 낮은 강조도로 표시한다.
- 뷰어에서는 `공유본 · 기기에 저장되지 않음`을 표시한다.
- 저장 실패 시에만 지속형 오류 배너와 파일 저장 대안을 제공한다.

### 3.10 모션 환경 설정 존중

**관찰**

- 버튼, 토스트, 색상 스와치 등에 전환 효과가 있으나 `prefers-reduced-motion` 대응이 없다.

**제안**

- `@media (prefers-reduced-motion: reduce)`에서 장식성 transition을 제거한다.
- 전술 패턴 재생은 사용자가 직접 시작한 핵심 기능이므로 유지하되, 단계 이동 시간을 짧게 설정할 수 있는 옵션을 제공한다.

## 5. 다음 검증에서 측정할 지표

- 첫 방문 사용자가 도움 없이 선수 이름을 수정하기까지 걸리는 시간
- 모바일에서 특정 선수의 지침을 찾아 수정하는 데 필요한 탭 수와 스크롤 거리
- 패턴 2단계를 만들고 미리보기까지 성공하는 비율
- 실수로 삭제한 뒤 복구하지 못한 세션 비율
- 공유 링크 진입 후 `가져오기`와 `내 스쿼드 열기`를 잘못 선택한 비율
- 키보드만으로 탭 이동, 모달 열기/닫기, 선수 편집이 가능한지 여부

## 6. 결론

현재 버전은 반응형과 비주얼 정비가 성공적으로 마무리된 상태다. 다음 단계는 화면을 더 꾸미는 것보다 **입력한 데이터를 안심하고 다룰 수 있게 만들고, 모바일에서 한 번에 보여주는 편집량을 줄이며, 숨은 제스처를 명시적 선택 흐름으로 바꾸는 것**이 효과가 크다.

가장 추천하는 다음 묶음은 `되돌리기 + 모바일 지침 접기 + 선수 선택 액션 바`다. 세 가지가 함께 적용되면 신규 사용자 학습성과 기존 사용자 작업 속도를 동시에 개선할 수 있다.

## 7. 구현 반영 현황

`codex/ui-ux-followups` 브랜치에서 다음 항목을 앱 코드에 반영했다.

- 선수·패턴·단계·경로 삭제와 단계 초기화 후 5초간 `되돌리기` 제공
- 모바일 개별 선수 지침 기본 접힘 및 작성 개수 표시
- 선수 클릭·Enter·Space 선택과 이름/색상/지침/삭제 빠른 작업 바
- 선수 요소의 키보드 포커스 및 접근 가능한 이름
- 앱 모드·스쿼드 탭의 방향키/Home/End 이동과 roving `tabindex`
- 공유·선수 시점 모달의 포커스 이동·트랩·복귀 및 배경 `inert`
- 토스트 `role="status"`/`aria-live` 적용
- 작은 보조 텍스트와 기본·공격 스쿼드 색상 대비 개선
- `prefers-reduced-motion` 대응

검증 결과 모바일 기본 화면 높이는 약 2,025px에서 1,313px로 감소했으며, 390px 필드 폭 355px와 데스크톱 2패널 배치를 유지했다. 선수 삭제 후 되돌리기, 탭 키보드 이동, 모달 포커스 순환과 복귀를 실제 브라우저에서 확인했고 콘솔 오류는 발생하지 않았다.
