# 반응형 UX 개편 + 전술 보드 테마 — 설계

날짜: 2026-07-18
상태: 승인됨 (범위: 전체 / 테마: 전술 보드)

## 배경·목표

이 서비스는 두 역할이 서로 다른 환경에서 쓴다.

- **작성자(감독)**: 주로 PC·태블릿 큰 화면에서 전술을 작성. 모바일 작성도 가능해야 함(보조).
- **열람자(팀원)**: 카톡 공유 링크를 모바일로 받아서 열람하는 것이 주 시나리오.

현재 구조의 문제:

1. 데스크톱에서도 480px 단일 컬럼 — 필드(660px 높이)에 밀려 지침·컨트롤이 화면 밖. 필드를 보며 작성 불가.
2. 필드가 480px 고정이라 좁은 폰에서 화면을 넘어감(좌우 패닝, 축소 렌더링 없음).
3. 공유 링크로 진입해도 편집기 UI 전체가 노출되고, 열람자가 조작하면 autoSave가 열람자의 기존 로컬 스쿼드를 덮어씀.
4. iOS Safari는 터치에서 `contextmenu` 이벤트를 발생시키지 않아 색상·삭제·선수 시점 메뉴 접근 불가.
5. slate-900 + #3b82f6 + 이모지 버튼 = 전형적인 "AI가 만든 Tailwind 기본값" 인상.

## 범위 (5단계, 단계별 PR)

### 1단계 — 필드 스케일링 (기반)

- **좌표계 480×660 유지.** 모든 JS 로직·저장 데이터·GIF 출력은 변경 없음.
- `#fieldWrapper`를 반응형으로: 폭은 컨테이너에 맞추고(`aspect-ratio: 480/660`), 내부 `#field`·`#patternCanvas`를 `transform: scale(s)` (`s = wrapperWidth / 480`, `transform-origin: top left`)로 렌더링.
- 스케일 상한: 데스크톱 2패널에서 필드를 최대 640px 폭까지 확대 허용.
- **드래그 좌표 수정**: HTML 필드의 선수 드래그는 clientX/Y 델타를 그대로 쓰므로 스케일 시 어긋남 → 델타를 `s`로 나눠 보정(`s`는 `field.getBoundingClientRect().width / FIELD_W`로 산출). 캔버스 쪽 `canvasCoords()`는 이미 rect 기반 변환이라 수정 불요.
- **캔버스 선명도**: `#patternCanvas` 백킹 스토어를 `s × devicePixelRatio` 크기로 재설정하고 `ctx.setTransform`으로 480×660 좌표계 유지. GIF 내보내기는 별도 오프스크린 480×660 캔버스를 쓰므로 영향 없음.
- **PNG 저장**: 캡처 직전 wrapper를 480px로 임시 고정 → html2canvas 캡처 → 복원 (transform 캡처 왜곡 회피).

### 2단계 — 뷰어 모드 (공유 링크 = 열람 전용)

`#s=` 해시로 진입하면 편집기 대신 **뷰어 모드**로 렌더링.

- 숨김: 팀 설정 카드, 인원 pills, 포메이션 select/초기화, 선수 추가, 지침 입력 textarea, 파일 저장/불러오기, 공유 텍스트 생성 버튼, 패턴 추가/삭제/단계 편집 버튼.
- 노출(열람 UI):
  - 상단: 팀명 + "공유받은 전술" 배지.
  - 탭 1 **스쿼드**: 기본/공격/수비 전환하며 배치 열람(드래그 비활성).
  - 탭 2 **전술 패턴**: 패턴 이전/다음 + 재생(미리보기) + GIF 저장. 편집 불가.
  - 탭 3 **지침**: 스쿼드별 팀 전체 지침 + 선수 선택 드롭다운으로 개인 지침·위치 확인(기존 선수 시점 보기 재활용).
- **저장 분리**: 뷰어 모드에서는 autoSave 완전 비활성 → 열람자의 기존 localStorage 보존.
- **내 스쿼드로 가져오기** 버튼: 명시적으로 누르면 확인 후 공유 상태를 localStorage에 저장하고 편집 모드로 전환. **내 스쿼드 열기** 버튼: 해시 제거 후 자기 데이터로 리로드.
- 드래그·더블클릭·컨텍스트 메뉴·인라인 편집 전부 비활성.

### 3단계 — 데스크톱 2패널 레이아웃 (≥1024px)

- 좌측: 필드 (sticky, 최대 640px 폭으로 확대) + 필드 직속 컨트롤(스쿼드 탭, 포메이션 바).
- 우측 패널(스크롤 독립): 앱 모드 탭 + 모드별 콘텐츠 — 스쿼드 모드에서는 팀 설정·지침 입력·공유, 패턴 모드에서는 패턴/단계 컨트롤, 전략 모드에서는 전략 폼(전략 모드는 필드 대신 전체 폭 사용 유지).
- 1024px 미만은 현재의 세로 스택 유지(1단계 스케일링 적용).
- DOM 이동 없이 CSS Grid `grid-template-areas`로 재배치를 우선 시도, 불가피한 경우에만 마크업 재구성.

### 4단계 — 전술 보드 테마

감독 전술판 컨셉. slate/파랑 탈피.

- 팔레트: 배경 `#141a16`(차콜 그린-블랙), 서피스 `#1c241e`, 보더 `#2c3830`, 본문 `#e8ede8`(초크 화이트), 뮤트 `#8fa096`, 포인트 `#b8e986`(라임), 경고 앰버, 위험 톤다운 레드.
- 필드 잔디 톤을 배경과 조화되게 조정, 초크 라인 유지.
- 타이포: 헤더에 콘덴스드 대문자 워드마크(웹폰트 1종, `font-display: swap`, 로컬 실행 시 시스템 폴백). 본문은 시스템 산세리프 유지.
- 버튼: 플랫 스타일, **이모지 전부 제거**, 필요한 곳만 인라인 SVG 아이콘. 색상 종류 축소(기본/포인트/위험 3계열).
- `alert`/`confirm`/`prompt` → 자체 모달·토스트로 교체.
- 스쿼드 탭(기본/공격/수비) 구분색은 유지하되 새 팔레트 톤으로 재조정.

### 5단계 — 터치 패리티·잔여 UX

- **롱프레스 메뉴**: pointer 이벤트 기반 500ms 롱프레스로 컨텍스트 메뉴 오픈(iOS 대응). 롱프레스 시 드래그와 충돌하지 않게 이동 임계값 처리.
- **화살표 삭제(터치)**: 패턴 모드에서 화살표 탭 시 삭제 팝오버 표시(우클릭은 유지).
- 포메이션 변경 시 커스텀 배치가 초기화됨을 confirm(자체 모달)으로 경고.
- 힌트 텍스트를 입력 방식(터치/마우스)에 맞게 표시.
- 탭 타깃 44px 미만인 컨트롤(포메이션 select, 패턴 nav 화살표) 확대.

## 범위 제외 (YAGNI)

- undo/redo — 별도 과제.
- 광고 삽입(`docs/ad-placement.md`), html2canvas CDN 하드닝 — 기존 후보 과제로 유지.
- 서버/계정/실시간 공유 — 없음. 단일 파일 유지.

## 검증

각 단계마다:

1. 인라인 스크립트 추출 후 `node --check`.
2. 헤드리스 Chrome 스크린샷: 390px(모바일)·768px(태블릿)·1440px(데스크톱) × 스쿼드/패턴/전략/뷰어 모드.
3. 헤드리스 하니스(iframe + `contentWindow.eval`)로 드래그 좌표 보정·뷰어 모드 저장 분리 단언.
4. claude-in-chrome 연결 시 실브라우저 확인.

## 리스크

- `transform: scale`과 html2canvas 조합 왜곡 → 캡처 시 스케일 임시 해제로 회피.
- 2패널 재배치 시 `.app-squad-only` 토글 로직과 충돌 가능 → `setAppMode()` 표시 로직을 CSS 클래스 기반으로 정리.
- 뷰어 모드 추가로 초기화 분기(`loadFromURL`/`loadFromLocalStorage`) 복잡도 증가 → 초기화 흐름을 단일 함수로 정리하고 하니스로 회귀 확인.
