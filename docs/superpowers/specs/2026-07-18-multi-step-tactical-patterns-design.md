# 다단계 전술 패턴 설계 (2026-07-18)

## 목표

기존 🎯 공격 패턴 탭(선수당 화살표 1개, 한 단계 이동)을 **전술 패턴**으로 확장한다.
약속된 움직임을 여러 단계로 나눠 순서대로 애니메이션하고, 공(볼)의 이동(패스·드리블)도 함께 표현한다.

## 선택한 접근: 누적 단계(Step) 모델

- 패턴은 여러 **단계(step)** 를 가진다. 각 단계 = 선수별 이동 화살표 묶음 + 공 이동(선택) 1개.
- 단계 N의 시작 위치 = 단계 N-1이 끝난 위치(누적). 재생 시 단계가 순서대로 이어진다.
- 대안이었던 자유 타임라인(웨이포인트) 모델은 순서 개념이 없어 전술 설명 목적에 부적합하여 기각.

## 데이터 모델

```js
pattern = {
  name: string,                       // ≤20자
  ballStart: { x, y },                // 공 시작 위치 (기본: 센터 240,330)
  steps: [ { moves: { playerId: {toX, toY} }, ball: {toX, toY} | null } ],
}
```

- 스냅샷(v:1 유지): `pat: [{ n, bs:{x,y}, s:[{ m:{id:{x,y}}, b:{x,y}|null }] }]`
- **하위호환**: 기존 `{n, m}` 형식은 복원 시 1단계 패턴으로 자동 마이그레이션.
  기존 공유 URL·`.sq`·localStorage 모두 그대로 열린다.
- 복원 시 sanitize: 이름 절단, 좌표 숫자 검증·필드 범위 클램프, 비배열 방어.

## 에디터 UI

- 패턴 바 아래 **단계 바**: `◀ 단계 1/3 ▶` `+ 단계` `🗑 단계` (마지막 1개는 삭제 불가)
- 캔버스는 "이전 단계들의 누적 종료 위치"를 기준으로 선수·공을 그리고, 현재 단계의 화살표만 편집
- 공 토큰(흰 원): 드래그 → 현재 단계의 공 이동 화살표(주황 #fb923c, 선수 화살표는 노랑 유지)
  - 15px 미만 드래그 → 공 시작 위치만 재배치
  - 화살표 끝점을 선수 위에 놓으면 그 선수의 단계 종료 위치로 스냅(침투하는 선수에게 패스)
- 우클릭으로 화살표 개별 삭제(선수·공 동일), `↺ 단계 초기화`는 현재 단계만 비움
- 탭 이름: 🎯 공격 패턴 → 🎯 전술 패턴

## 매치 전략 연동

- 지침서의 선수별 이동 설명: `[패턴명 2단계] 전방 침투 이동` 형식으로 단계 반영
- 요약의 활성 패턴 판정: 어느 단계든 이동(선수 또는 공)이 있으면 활성
