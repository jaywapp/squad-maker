# 스쿼드 메이커 — 클로즈드 베타 준비 정리

- 작성일: 2026-07-21
- 대상: 운영자(팀 관리자) 본인
- 참가자: 소속 축구팀 임원진 (개발·프로그래밍과 무관한 실사용자)
- 성격: 수익화 본격 착수 전, **실사용자 반응을 직접 확인하기 위한 소규모 클로즈드 베타**

> 이 문서는 "무엇이 준비됐고, 운영자가 무엇을 해야 하는가"를 정리한다.
> 참가자에게 나눠 줄 사용법은 [사용자 가이드](./2026-07-21-user-guide.md)를 따로 참고한다.

---

## 1. 왜 지금 베타인가

수익화 계획서([링크](./2026-07-20-monetization-implementation-plan.md))는 Gate A 통과 전까지
인증·클라우드·결제를 만들지 말라고 못 박고 있다. 즉 지금 필요한 것은 **기능 추가가 아니라
"사람들이 실제로 이 도구를 쓰는가"에 대한 근거**다.

참가자가 비개발자이므로, 코드나 콘솔이 아니라 **손에 쥐여 주면 바로 쓸 수 있는 상태**와
**막혔을 때 말할 수 있는 창구**가 준비의 핵심이다. 이번 준비는 그 두 가지에 집중했다.

---

## 2. 앱에 반영한 것 (준비 완료)

| 항목 | 내용 | 위치 |
|---|---|---|
| 도움말 | 상단 `도움말` 버튼 → 빠른 시작 안내 모달(스쿼드·지침·공유·저장 4단계) | `index.html` topbar / `#helpModal` |
| 의견 보내기 | 상단 `의견 보내기` 버튼 → 구글 폼을 새 탭으로 염 | `openFeedback()` |
| 진단 정보 복사 | 도움말 안 `진단 정보 복사` → 버전·화면·브라우저 등 **환경 정보만** 복사 (이름·지침 등 개인정보 미포함) | `buildDiagnostics()` |
| 버전 표시 | 상단 좌측에 `beta 1 (2026-07-21)` 상시 표기 → 참가자가 "무슨 버전"인지 말할 수 있음 | `APP_VERSION` |
| 뷰어 모드 안내 | 공유 링크로 연 참가자도 도움말·의견 버튼을 그대로 사용 가능 | topbar는 편집·뷰어 공통 |

- **UI 이모지 없음** 규칙을 지켜 버튼·라벨은 텍스트만 사용했다.
- 도움말·의견 창구는 기존 무료 기능에 아무 제한을 걸지 않는다(수익화 계획 §2.1 무료 불변 조건 유지).
- 진단 정보 정책: 선수·팀 이름, 팀/선수 지침, 스쿼드 내용은 **어떤 경우에도 수집·복사하지 않는다.**
  복사되는 값은 앱 버전, 화면/창 크기, 편집·뷰어 여부, 현재 화면, 저장 데이터 유무, 브라우저 종류, 시각뿐이다.

## 3. 분석(Umami)은 베타 기간 끈다

- 참가자 규모가 작아 **통계보다 직접 관찰·인터뷰가 훨씬 정확**하다.
- 계획서상 분석 활성화 전 필요한 `docs/privacy-data-map.md`가 아직 없다.
- 따라서 `SQUAD_MAKER_ANALYTICS_ID`는 **비운 상태로 유지**한다. 어댑터는 ID가 없으면 아무 요청도 보내지 않는다.
- 관련 방어(전송 본문 가드·화이트리스트)는 코드에 그대로 있으므로, 나중에 켤 때 ID만 주입하면 된다.

> 결정: **끄고 진행.** 근거·전환 조건은 [ADR 0001](./adr/0001-analytics-provider.md)과 이 문서 §6을 따른다.

## 4. 운영자가 배포 전에 할 일 (체크리스트)

- [ ] **구글 폼 생성** — 아래 §5 질문 세트 참고. 응답을 스프레드시트로 받도록 설정.
- [ ] **폼 링크를 앱에 주입** — `index.html`의 베타 설정 블록에서 아래 한 줄의 주석을 풀고 링크를 넣는다:
  ```html
  <script>
    window.SQUAD_MAKER_FEEDBACK_URL = 'https://forms.gle/실제_폼_링크';
  </script>
  ```
  이 값은 시크릿이 아니므로 커밋해도 된다. 넣지 않으면 `의견 보내기`는
  "준비 중"이라는 안내만 띄우고 아무 일도 하지 않는다(안전한 폴백).
- [ ] **feature 브랜치 → PR → main 머지** → GitHub Pages 자동 배포 확인.
- [ ] 배포 후 실제 URL을 휴대폰에서 열어 도움말·의견 버튼·공유 링크를 한 번씩 눌러 본다.
- [ ] 참가자에게 **라이브 URL + [사용자 가이드](./2026-07-21-user-guide.md)** 를 함께 전달.
- [ ] 관찰 기간(권장 2~3주)과 인터뷰 대상(3~5명)을 미리 정한다.

---

## 5. 구글 폼 질문 세트 (제안)

짧게, 비개발자가 1~2분에 끝낼 수 있게 구성한다.

1. **어떤 상황이었나요?** (한 줄) — 예: "선수 위치를 옮기려는데 안 움직였어요"
2. **어느 화면이었나요?** (객관식: 스쿼드 / 전술 패턴 / 매치 전략 / 공유받은 링크 열람 / 모르겠음)
3. **무엇을 하려고 했나요?** (서술)
4. **어떻게 됐나요?** (서술)
5. **기기·브라우저** (객관식: 아이폰 사파리 / 안드로이드 크롬 / PC 크롬 / 기타)
6. **이 기능이 있으면 좋겠다 싶은 게 있나요?** (선택, 서술)
7. (선택) 진단 정보를 복사해 오셨다면 여기에 붙여넣어 주세요. (장문 텍스트)

> 개인정보(이름·연락처)는 폼에서 **묻지 않는다.** 인터뷰가 필요하면 별도로 구두 요청한다.

---

## 6. 관찰 지표 (수동)

분석을 끈 대신, 다음을 **직접·인터뷰로** 확인한다.

| 보고 싶은 것 | 확인 방법 |
|---|---|
| 가입 없이 스쿼드를 끝까지 만드는가 | 참가자에게 직접 물어보거나 옆에서 관찰 |
| 어디서 막히는가 | 의견 폼 응답 + 도움말 열람 여부(구두) |
| 공유 링크가 실제로 열리는가 | 링크를 받은 사람에게 확인 |
| 다시 쓰고 싶은가 | 2~3주 뒤 재사용 의향 인터뷰 |

Gate A 판정 기준(활성 제작자 비율·완성 세션 수 등)은 베타 규모가 작아 이 단계에서 **통계로 세지 않는다.**
베타의 목적은 "숫자 충족"이 아니라 "무료 도구로서 매력이 있는지, 어디를 고쳐야 하는지"를 아는 것이다.

## 7. 롤백 · 리스크

- 이번 변경은 **추가만** 했고 기존 무료 기능 로직을 건드리지 않았다. 문제가 생기면 해당 PR만 되돌리면 된다.
- 회귀 계약 테스트(`npm test`)가 배포를 차단한다 — 편집·저장·PNG/GIF·공유·뷰어·`.sq` 왕복이 깨지면 머지되지 않는다.
- 의견 폼 URL 미설정 시에도 앱은 정상 동작한다(버튼이 안내만 띄움).
