# 📢 스쿼드 메이커 — 광고 삽입 포인트 문서

> 작성: Team Jaywapp / 대상: `index.html` (단일 파일 정적 웹앱, GitHub Pages 호스팅)
> 목적: 무료 유틸리티 앱의 광고 수익화를 위한 **삽입 위치·포맷·UX 영향**을 정의한다.

---

## 1. 전제 조건

| 항목 | 내용 |
|------|------|
| 호스팅 | GitHub Pages (정적) → **클라이언트 사이드 광고만 가능** (Google AdSense 등) |
| 앱 구조 | 단일 `index.html`, 빌드 도구 없음 → 광고 스크립트는 `<head>` 또는 `<body>` 말미에 직접 삽입 |
| 핵심 인터랙션 | 필드 위 선수 **드래그/더블클릭/우클릭** → 이 영역을 광고로 가리면 안 됨 |
| 사용 흐름 | 짧은 세션(라인업 1~2개 작성 → 저장/공유 후 이탈)이 많음 → **결과 산출 시점**이 광고 노출 골든타임 |
| 모바일 비중 | 카톡/단톡방 공유가 주 용도 → 모바일 트래픽 높음 → 반응형 광고 필수 |

### 광고 배치 5원칙
1. **인터랙션 영역(필드) 침범 금지** — 드래그·클릭 정확도를 해치면 이탈로 직결
2. **결과 산출 직후 노출** — 이미지 저장/GIF 생성/공유 완료 시점이 수용성 최고
3. **레이아웃 시프트(CLS) 방지** — 광고 슬롯 높이를 CSS로 예약
4. **모달 안 인터랙션 방해 금지** — 닫기·복사 버튼 위/아래 안전 여백 확보
5. **선택적 비활성** — 공유 링크(`?readonly`)로 진입한 외부 사용자에게는 광고 최소화(첫인상 보호)

## 2. 우선순위별 삽입 포인트

### 🥇 P1 — 푸터 앵커 광고 (필수, 최우선)
- **위치**: `<p class="hint">`(index.html:460) 아래, `</main>` 직전 또는 직후
- **포맷**: 반응형 디스플레이 배너 (모바일 320×100 / 데스크톱 728×90)
- **근거**: 모든 사용자가 스크롤 끝에서 보게 되는 안정적 인벤토리. 인터랙션과 분리됨.
- **UX 영향**: 낮음. `max-width:480px` 컨테이너 흐름에 자연스럽게 편입.
- **구현 앵커**:
  ```html
  <!-- index.html: </main> 바로 아래 -->
  <div class="ad-slot ad-footer" aria-hidden="true"><!-- AdSense ins --></div>
  ```
  CSS: `.ad-footer{width:100%;max-width:480px;min-height:100px;margin:16px auto;}`
  (공통 `max-width:480px` 규칙 셀렉터 그룹에 합류시키면 정렬 일관성 확보)

### 🥈 P2 — 공유 텍스트 모달 내 광고 (고전환)
- **위치**: 공유 모달(`#shareModal`, index.html:502) 내부 — `share-actions`(복사/저장 버튼)와 `share-divider` 사이
- **포맷**: 인-모달 사각형 (300×250 medium rectangle)
- **근거**: "단톡방 공유 텍스트 생성"은 **명확한 작업 완료 신호**. 사용자가 텍스트를 복사하는 동안 자연 노출. 모달 체류 시간이 길다.
- **UX 영향**: 중간. 복사 버튼과 닫기 버튼 사이 충돌 주의 → 광고 위/아래 12px 이상 패딩 필수.
- **구현 앵커**: `#shareModal .share-box` 내 `.share-divider` 직전.

### 🥉 P3 — 이미지/GIF 저장 완료 인터스티셜
- **위치**: `saveImage()` (📸 이미지 저장), GIF 저장 콜백 **완료 직후** 동적 삽입
- **포맷**: 토스트 확장형 또는 가벼운 인터스티셜 카드 (자동 닫힘 X, 사용자 닫기)
- **근거**: PNG/GIF 다운로드는 핵심 전환 액션. 결과물을 얻은 만족 시점 = 광고 수용성 최고점.
- **UX 영향**: 중간~높음. **전면 인터스티셜은 지양**, 다운로드를 막지 않는 비차단(non-blocking) 형태 권장. 기존 `showToast()`(index.html:789, `.toast` 클래스) 패턴을 확장해 스폰서 메시지 슬롯으로 재사용 가능.
- **빈도 제한**: 세션당 1~2회 (저장 반복 시 매번 노출 금지).

### P4 — 상단 리더보드 (선택)
- **위치**: `<h1>`(index.html:397)과 `#readonlyBanner`/`team-setup` 사이
- **포맷**: 반응형 리더보드 (모바일 320×50 / 데스크톱 728×90)
- **근거**: 최초 진입 시 노출(above the fold). 단, 첫인상·로딩 체감 저하 위험.
- **UX 영향**: 높음(주의). 페이지 진입 즉시 광고는 이탈률을 높일 수 있어 **A/B 테스트 후 결정**. 공유 링크 진입자에게는 숨김 권장.

### P5 — 데스크톱 사이드 레일 (선택, 데스크톱 전용)
- **위치**: `@media (min-width:960px)` 환경에서 `body`(max-width:960px, 중앙 정렬) 좌/우 여백
- **포맷**: 고정 스카이스크래퍼 (160×600)
- **근거**: 데스크톱에서 본문이 480px로 좁아 좌우 여백이 큼 → 본문 침범 없이 인벤토리 확보.
- **UX 영향**: 낮음(데스크톱 한정). 모바일에서는 자동 비표시.
- **주의**: `position:fixed`로 배치하되 960px 미만에서는 `display:none`.

## 4. 구현 가이드 (요약)

1. **스크립트 로드**: `<head>`에 AdSense `async` 스크립트 1회 삽입 (기존 html2canvas/gif.js와 동일 패턴).
2. **슬롯 예약**: 각 `.ad-slot`에 `min-height` 지정 → CLS 0 유지.
3. **반응형**: 공통 `max-width:480px` 컨테이너 규칙(index.html CSS 상단)을 광고 슬롯에도 적용해 본문 정렬과 일치.
4. **읽기전용 모드 분기**: `loadFromURL()`로 진입한 공유 뷰어(`#readonlyBanner.visible`)에서는 P1만 노출, P3/P4 억제.
5. **빈도/동의**: 저장 인터스티셜은 `localStorage`(기존 `LS_KEY='squad-maker-v1'` 옆에 별도 키)로 세션 노출 횟수 관리. GDPR/개인정보 동의 배너는 EU 트래픽 발생 시 추가.
