## 1. 결론

제안한 방향은 타당하다. 스쿼드 메이커의 수익화 중심은 **스쿼드를 그리는 행위가 아니라, 만든 전술을 계속 저장·재사용하고 팀원별로 전달하며 이해 여부를 관리하는 과정**이어야 한다.

권장 구조는 다음과 같다.

1. **비회원 무료**: 스쿼드 생성과 기본 편집을 완전히 개방해 검색·입소문 유입을 확보한다.
2. **무료 회원**: 클라우드 저장, 영구 공유 링크, 기기 간 동기화 등 계정을 만들 이유를 제공한다.
3. **Pro 유료**: 여러 팀의 클라우드 보관, 고급 애니메이션 내보내기, 선수별 비공개 전술 전송, 열람 확인과 버전 관리에 과금한다.
4. **Club 유료**: 여러 코치·팀을 관리하는 조직 기능은 사용자 수요가 확인된 뒤 추가한다.

핵심 수정 제안은 두 가지다.

- **현재 무료 기능은 회수하지 않는다.** 무제한 로컬 패턴, GIF, PNG, `.sq` 파일과 로컬 선수 명단은 기존과 같이 비회원도 무료로 사용한다.
- **유료 상품은 새 가치에 과금한다.** 여러 팀의 클라우드 저장, 고해상도·영상 출력, 선수별 브리핑과 열람 확인처럼 서버와 협업이 필요한 기능을 Pro로 제공한다.

## 4. 권장 상품 구조

### 4.1 Free Guest — 비회원 무료

목표는 방문자가 1분 안에 결과물을 만들고, 제품 이름이 포함된 결과를 다른 사람에게 전파하게 하는 것이다.

- 모든 인원제 포메이션 생성과 선수 배치
- 팀·선수 기본 지침 작성
- 브라우저 로컬 자동 저장: 최근 작업 1개
- 로컬 선수 명단 저장과 재사용
- 현재와 동일한 PNG 및 GIF 저장
- 공유 링크 열람과 내 로컬 스쿼드로 가져오기
- 로컬 전술 패턴·단계 생성과 미리보기: 제한 없음
- `.sq` 파일 저장/불러오기

현재처럼 공유 상태를 URL에 담는 방식은 서버 비용이 거의 들지 않고 다른 사용자를 데려오는 핵심 루프이므로, **기존 링크 생성과 열람은 항상 무료·비회원**이어야 한다. 회원에게는 링크 이름 변경·취소·열람 통계처럼 서버에 저장해야 가능한 관리 기능을 추가한다.

### 4.2 Free Account — 무료 회원

목표는 결제를 강요하기 전에 사용자의 팀과 전술을 제품 안에 축적하는 것이다.

- Guest의 모든 기능
- 클라우드 저장: 팀 1개, 스쿼드/전술 문서 최대 5개
- 기기 간 동기화
- 관리형 공유 링크 최대 5개
- 링크 제목과 설명 편집
- 공유 문서 복제와 최근 버전 복구: 7일
- 기존 로컬 패턴·PNG·GIF 기능 유지

회원가입 시점은 첫 진입이 아니라 다음 행동 직전에 둔다.

- 두 번째 전술을 클라우드에 저장하려 할 때
- 공유 링크를 다시 찾거나 이름을 바꾸려 할 때
- 다른 기기에서 이어서 작업하려 할 때
- 공유 링크를 취소하거나 열람 현황을 확인하려 할 때

### 4.3 Pro Coach — 핵심 유료 상품

목표는 코치가 매주 반복하는 전술 준비와 전달 시간을 줄이는 것이다.

- Free Account의 모든 기능
- 클라우드 팀 최대 5개, 선수 명단과 전술 문서 무제한
- 여러 선수 명단의 클라우드 저장·재사용 및 시즌별 보관
- 고해상도 PNG/PDF와 MP4 등 고급 영상 내보내기
- 영상 해상도·재생 속도·팀 브랜딩 설정
- **선수별 비공개 전술 링크**: 해당 선수의 위치·개별 지침·관련 애니메이션만 표시
- 선수별 일괄 전송 링크 생성과 복사
- 열람 여부·최근 열람 시각 확인
- 전술 수정 이력과 30일 버전 복구
- 팀 템플릿 복제, 광고 제거
- 향후 경기별 전술 묶음과 읽음 확인 리마인드

`개별 전술 전송`은 단순히 텍스트를 나눠 보내는 기능보다 다음 형태일 때 결제 가치가 커진다.

1. 코치가 경기 전술을 완성한다.
2. 앱이 선수별 전용 링크를 자동 생성한다.
3. 선수는 가입 없이 자신의 위치·지침·애니메이션만 본다.
4. 코치는 누가 확인했는지 한 화면에서 본다.

즉, 유료 기능의 이름도 `개별 전술 전송`보다 **선수별 전술 브리핑**으로 정의하는 것을 권장한다.

### 4.4 Club — 수요 검증 후 출시

- 코치 좌석과 여러 팀 통합 관리
- 클럽 공용 선수·포메이션·훈련 템플릿
- 코치별 편집/열람 권한
- 클럽 로고와 브랜드 적용
- 감사 로그, 데이터 내보내기, 우선 지원
- 연간 청구와 세금계산서 등 조직 결제 지원

초기에는 기능을 먼저 만들지 말고, Pro 사용자 중 팀 3개 이상을 운영하거나 코치 초대를 요청하는 고객에게 수동 상담을 제공해 수요를 확인한다.

- 카드 등록 없는 14일 Pro 체험보다, **회원가입 즉시 7일 Pro 체험** 또는 특정 유료 결과물을 1회 체험시키는 방식을 우선 시험한다.

- 영구 이용권은 장기 서버·지원 비용과 맞지 않으므로 제공하지 않는다.

초기 손익분기는 대표자 인건비를 제외한 **현금성 운영비 회수 기준**으로 계산한다. 실제 가격·PG 계약·인프라가 확정되기 전의 내부 가설이며, 부가가치세 포함 소비자가와 결제 수수료 3%를 가정했다.

## 7. 광고 수익과 구독 수익의 관계

저장소에는 광고 배치안(`docs/ad-placement.md`)이 있지만, 현 단계의 1순위는 광고 삽입이 아니라 **활성 사용자와 Pro 결제 의향 검증**이다.

광고는 상당한 트래픽이 생기기 전에는 수익 기여가 작고 편집 도구의 집중도를 떨어뜨릴 수 있다. 적용한다면 다음 원칙을 권장한다.

- 필드와 편집 과정에는 광고를 두지 않는다.
- 무료 결과 화면 또는 페이지 하단에만 낮은 빈도로 노출한다.
- 공유받은 선수의 첫 열람 화면에는 광고를 최소화한다.
- Pro는 항상 광고가 없다.
- 축구 용품점·구장·리그 등 문맥에 맞는 직접 스폰서십이 가능해지면 일반 배너보다 우선 검토한다.

## 8. 수익화에 필요한 제품·운영 구조 변화

### 8.1 기술 요건

현재 앱은 단일 정적 HTML과 LocalStorage, URL 인코딩 공유로 동작한다. 무료 회원과 유료 기능을 제공하려면 다음 백엔드 기능이 새로 필요하다.

1. **인증**: 이메일 또는 소셜 로그인, 비회원 데이터의 안전한 계정 이전
2. **데이터 저장**: 사용자·팀·선수·전술·버전·공유 링크 데이터 모델
3. **권한**: 소유자/코치/선수, 공개/비공개 링크, 만료와 취소
4. **결제**: 월·연 구독, 상태 동기화, 실패·해지·환불 처리
5. **파일 처리**: 서버 또는 클라이언트 내보내기 정책, 저장 용량 제한
6. **이벤트 측정**: 가입·활성화·공유·페이월·결제 퍼널
7. **개인정보 보호**: 최소 정보 수집, 선수 링크의 추측 방지, 삭제·내보내기 절차

기존 URL 공유는 하위 호환을 유지한다. 유료 비공개 공유만 서버의 짧은 토큰 링크로 추가하고, 기존 공개 링크를 갑자기 유료화하지 않는다.

### 8.2 한국 사업·법무·결제 요건

다음 내용은 법률·세무 자문이 아니라 출시 준비 체크리스트다. 실제 적용 여부는 판매 방식과 사업 규모를 기준으로 전문가 또는 관할 기관에 확인한다.

1. **사업자등록**: 유료 서비스 개시 전에 업종·과세 유형과 사업자등록 절차를 확인한다.
2. **통신판매업 신고**: 온라인 구독 판매는 원칙적으로 신고 대상이지만 거래횟수·규모에 따른 면제 기준이 있으므로, 출시 시점의 적용 여부를 확인하고 필요하면 신고한다. ([전자상거래법 제12조](https://www.law.go.kr/LSW/lsSideInfoP.do?docCls=jo&joBrNo=00&joNo=0012&lsiSeq=282793&urlMode=lsScJoRltInfoR))
3. **전자상거래 고지**: 사업자 정보, 가격·결제주기, 자동 갱신, 청약철회·해지·환불 조건과 이용약관을 결제 전에 명확히 표시한다.
4. **PG 선정**: 국내 사업자가 직접 계약 가능한 국내 PG/결제대행과 정기결제 지원 여부를 우선 비교한다. Stripe는 한국이 계정 개설 지원 국가 목록에 없지만, 지원 국가의 사업자는 한국 카드·KRW·정기결제를 처리할 수 있으므로 법인 소재지와 정산 구조까지 함께 판단한다. ([Stripe 지원 국가](https://stripe.com/global), [한국 결제수단 문서](https://docs.stripe.com/payments/countries/korea?locale=en-GB))
5. **개인정보 처리방침**: 수집 목적·항목·보유기간·제3자 제공·파기·정보주체 권리와 안전조치를 문서화하고 공개한다. ([개인정보 보호법 제30조](https://law.go.kr/LSW/lsLinkCommonInfo.do?chrClsCd=010202&lsJoLnkSeq=1020398583))
6. **선수 정보의 처리 근거**: 코치가 입력한 이름도 다른 정보와 결합해 개인을 식별할 수 있으면 개인정보가 될 수 있다. 수집 최소화, 가명/등번호 사용 옵션, 팀 삭제 시 일괄 파기와 접근 통제를 설계한다.
7. **만 14세 미만 아동**: 아동이 직접 가입하거나 동의를 근거로 개인정보를 수집하는 흐름에서는 법정대리인 동의 요건을 검토한다. 코치가 대신 입력하는 선수 정보도 수집 경로와 처리 근거가 다르므로 별도 법률 검토 없이 일괄 동의를 전제하지 않는다.

출시 게이트는 두 단계로 나눈다.

- **무료 계정·클라우드 저장 전**: 개인정보 처리방침, 이용약관, 데이터 삭제 요청, 보안·백업 절차 완료
- **유료 베타 전**: 사업자·통신판매업 해당 여부 확인, PG 계약, 구독 고지, 해지·환불·결제 실패 대응 완료

## 9. 추천 출시 로드맵

아래 일정은 **주당 10~15시간의 사이드 프로젝트**를 가정한 범위다. 외부 심사, PG 계약과 법무 검토 기간은 별도이며, 각 단계는 앞 단계 지표가 기준을 충족할 때만 진행한다.

### 0단계 — 결제 전 수요 측정 (2~4주)

- 개인정보 수집을 최소화한 이벤트 분석 도구 선정·도입
- 정적 사이트에 적용하기 쉬운 Plausible Cloud와 Umami Cloud/자체 호스팅을 우선 비교하고, 비용·운영 부담·데이터 위치·동의 요건을 확인한다.
- `클라우드 팀 저장`, `선수별 전술 브리핑`, `고급 영상 내보내기` 버튼에 설명형 사전 안내 페이지 연결
- 사용자가 관심 기능과 예상 사용 빈도를 남길 수 있게 한다.
- 실제 결제는 받지 않고 클릭률과 이메일 대기 명단을 측정한다.

### 1단계 — 무료 계정과 클라우드 저장 (6~10주)

- 로그인과 비회원 로컬 데이터 마이그레이션
- 팀 1개·문서 5개 클라우드 저장
- 영구 공유 링크와 링크 관리 화면
- 가입 퍼널 및 1주 재방문 측정

### 2단계 — Pro 최소 상품 (6~10주)

- 구독 결제
- 무제한 저장과 선수 명단 재사용
- 고해상도 PNG/PDF와 고급 영상 내보내기
- 구독 상태·해지·복구 처리

### 3단계 — 차별화 기능 (8~12주)

- 선수별 비공개 브리핑 링크
- 선수별 관련 장면/지침 구성
- 열람 확인과 미확인 선수 표시
- 전술 버전 이력

### 4단계 — Club 탐색

- Pro 고객 인터뷰로 코치 좌석·다팀 관리 수요 확인
- 먼저 수동 견적과 온보딩으로 3~5개 조직을 확보
- 반복 요청만 제품 기능으로 전환

## 10. 핵심 지표와 의사결정 기준

방문자 수보다 `전술을 만들어 실제로 전달했는가`를 핵심 가치 지표로 삼는다.

### 핵심 가치 지표

- 주간 **공유된 전술 문서 수**
- 주간 **전술을 공유한 활성 코치 수**
- 선수별 브리핑의 **열람 완료율**

3단계 이전에는 선수별 열람 완료율을 측정할 수 없으므로 **공유 링크 생성 후 7일 이내 1회 이상 열람된 비율**을 초기 대리 지표로 사용한다. 선수별 브리핑 출시 후 `첫 공유 → 전원 확인 완료 시간`을 북극성 지표로 전환한다.

### 퍼널 지표

| 단계 | 이벤트 예시 | 초기 판단 기준 |
|---|---|---:|
| 생성 | 선수 이름 1명 이상 수정 또는 포메이션 변경 | 방문자의 35% 이상 |
| 완성 | 이미지·링크·파일 중 하나 생성 | 생성자의 40% 이상 |
| 공유 | 공유 링크 복사 | 완성자의 30% 이상 |
| 가입 | 클라우드 저장 또는 영구 링크를 위해 가입 | 공유자의 15% 이상 |
| Pro 관심 | 유료 기능 안내 확인 | 주간 활성 회원의 10% 이상 |
| 결제 | 체험 또는 Pro 구매 | 주간 활성 회원의 2~5% 가설 |
| 유지 | 결제 8주 후 활성 구독 | 60% 이상 목표 |

수치는 업계 평균을 주장하는 값이 아니라, 초기 제품 의사결정을 위한 내부 가설이다. 실제 유입 채널과 사용자군에 따라 조정한다.

### 반드시 기록할 이벤트

- `squad_started`, `squad_completed`
- `image_exported`, `share_link_created`, `share_link_opened`
- `account_signup_started`, `account_signup_completed`
- `cloud_save_limit_reached`, `managed_share_limit_reached`
- `advanced_export_previewed`, `player_briefing_previewed`
- `paywall_viewed`, `checkout_started`, `subscription_started`
- `subscription_cancelled`와 취소 사유

## 11. 검증할 핵심 가설

1. 사용자는 단발성 스쿼드 작성보다 선수 명단 재사용에 돈을 낸다.
2. 애니메이션을 본 코치는 정적 이미지 사용자보다 Pro 의향이 높다.
3. 선수별 링크와 열람 확인은 단순 GIF 내보내기보다 유지율에 더 큰 영향을 준다.
4. 무료 공유 결과에 선택적으로 표시한 제품 출처 링크는 이탈을 크게 늘리지 않으면서 신규 유입을 만든다.
5. 팀당 가격은 개인 코치당 가격보다 실제 결제 주체와 더 잘 맞는다.

각 가설은 기능 개발 전 인터뷰 5~8건, 설명형 버튼 클릭률, 대기 명단 또는 선결제 의향으로 순서대로 검증한다.

## 12. 주요 위험과 대응

| 위험 | 영향 | 대응 |
|---|---|---|
| 기존 무료 기능을 회수해 사용자가 이탈 | 바이럴·신뢰 감소 | 로컬 패턴·GIF·PNG·명단을 계속 무료 제공하고 신규 서버·협업 가치에만 과금 |
| 기능 수만 늘고 반복 사용이 낮음 | 개발비 대비 매출 부족 | 저장·재사용·열람 확인 순으로 검증 |
| 팀원에게 가입을 요구 | 공유 열람률 저하 | 선수는 가입 없이 비공개 링크 열람 |
| 계정 도입 중 로컬 데이터 손실 | 신뢰 훼손 | 가입 전 스냅샷, 명시적 병합, 되돌리기 제공 |
| 가격이 동호회 예산보다 높음 | 구매 포기 | 월/연 병행, 초기 고객 가격 실험 |
| 개인 지침 링크 노출 | 개인정보·팀 전술 유출 | 긴 토큰, 링크 취소·만료, 검색 차단, 최소 정보 수집 |

## 13. 최종 제안

스쿼드 메이커는 **무료 전술 그림 도구**에서 **경기 전 브리핑을 완성하는 코치 워크플로우**로 확장해야 한다.

가장 먼저 만들 수익 기능은 다음 세 가지다.

1. **선수 명단과 전술의 클라우드 저장·재사용**
2. **고해상도·영상 등 전문 애니메이션 내보내기**
3. **선수별 전술 브리핑 링크와 열람 확인**

이 중 장기 차별화 요소는 세 번째다. 포메이션 작성과 GIF 생성은 경쟁자가 쉽게 제공할 수 있지만, 코치가 한 번 만든 전술을 선수별로 자동 분리해 보내고 확인 여부까지 관리하는 경험은 스쿼드 메이커만의 명확한 결제 이유가 될 수 있다.

따라서 첫 수익화 목표는 광고 수익이나 기능 잠금률이 아니라 다음 문장으로 정의하는 것을 권장한다.

> **코치가 경기 전 전술을 준비하고 모든 선수가 확인할 때까지 걸리는 시간을 절반으로 줄인다.**
